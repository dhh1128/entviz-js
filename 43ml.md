# autoLabel resolver on the corpus (TrustAssumption) object
kind: idea
created: 2026-07-16T07:17Z

- 2026-07-16T07:17Z A function property on the TrustAssumption/corpus object that takes an entropy value and looks it up somewhere (registry, address book, KEL alias table, etc.); if found, returns a human label to render into the entviz pill's label slot. Distinct from the mnemonic auto-label (this.i mmtxrg4w), which is a DETERMINISTIC, value-derived string built only from cells the entviz itself shows — autoLabel is a host-supplied LOOKUP that can surface a real name the value maps to. Open design questions: (1) probably async (lookup may hit I/O) and lazy (resolve on first display / on demand, not eagerly for every pill); (2) how the resolved label ranks vs an explicit host `label` and vs the mnemonic; (3) caching + invalidation of resolved labels; (4) security posture — a looked-up name is host-asserted, still corpus-gated, and must not be mistaken for verification (rule-out, never rule-in / uibwfl47). Gated by the TrustAssumption object (ujdwjtex) like the other recognition aids.
